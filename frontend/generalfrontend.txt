## Descripción
Proyecto frontend desarrollado con Angular 20.3.0 para la aplicación de aprendizaje de programación en C tipo Duolingo.

## Tecnologías Principales
- **Framework**: Angular 20.3.0
- **Lenguaje**: TypeScript 5.9.2
- **Gestor de Paquetes**: npm
- **Herramientas de Desarrollo**: Angular CLI 20.3.6
- **Testing**: Jasmine 5.9.0 + Karma 6.4.0
- **Formateo**: Prettier (configurado)

## Configuración del Proyecto
- **Nombre**: frontend
- **Prefijo de Componentes**: app
- **Puerto de Desarrollo**: 4200 (por defecto)
- **Puerto de Testing**: 9876

## 4.1 Componente Principal (app.ts)
```typescript
import { Component, signal } from '@angular/core';
import { RouterOutlet } from '@angular/router';

@Component({
  selector: 'app-root',
  imports: [RouterOutlet],
  templateUrl: './app.html',
  styleUrl: './app.css'
})
export class App {
  protected readonly title = signal('frontend');
}
```

**Características:**
- Usa Angular Signals (nueva funcionalidad de Angular 16+)
- Importa RouterOutlet para navegación
- Template y estilos separados en archivos HTML y CSS

## 4.2 Configuración de la Aplicación (app.config.ts)
```typescript
import { ApplicationConfig, provideBrowserGlobalErrorListeners, provideZoneChangeDetection } from '@angular/core';
import { provideRouter } from '@angular/router';
import { routes } from './app.routes';

export const appConfig: ApplicationConfig = {
  providers: [
    provideBrowserGlobalErrorListeners(),
    provideZoneChangeDetection({ eventCoalescing: true }),
    provideRouter(routes)
  ]
};
```

**Características:**
- Configuración moderna con providers functions
- Manejo global de errores
- Detección de cambios optimizada
- Configuración de rutas

- **Standalone Components**: Sin necesidad de NgModules

## 10.2 Mejores Prácticas
- Usar TypeScript estricto
- Implementar lazy loading para rutas
- Usar OnPush change detection strategy
- Implementar error handling global
- Usar interceptors para autenticación
- Implementar loading states
- Usar reactive forms para formularios

## 10.3 Testing
- Configuración Karma + Jasmine lista
- Tests unitarios para componentes
- Tests de servicios
- Tests de integración recomendados

## 10.4 Performance
- Lazy loading de módulos
- OnPush change detection
